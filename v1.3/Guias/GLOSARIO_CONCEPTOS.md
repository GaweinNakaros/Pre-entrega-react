## 1. Hooks de React

> **HOOKS:** Funciones especiales que permiten "engancharse" a caracter√≠sticas de React. Siempre empiezan con "use" (useState, useEffect, useContext, etc.). Solo se pueden usar dentro de componentes funcionales.

### useState

Maneja estado local en un componente. Retorna un array con 2 elementos: `[valorActual, funcionParaActualizarlo]`

```javascript

  // Sintaxis de destructuring de arrays

  //     ‚Üë          ‚Üë                        ‚Üë
  //   valor     funci√≥n               valor inicial
  //  actual   actualizadora
  
  // Para actualizar el estado:
  setContador(contador + 1); // Nuevo valor directamente
  
  // O con funci√≥n (cuando el nuevo valor depende del anterior):
  setContador(prevContador => prevContador + 1);

```

**üìå Puntos clave:**
- Primer elemento: valor actual del estado
- Segundo elemento: funci√≥n para actualizar el estado
- El argumento es el valor inicial del estado

### useEffect

## 2. Context API

> Context permite pasar datos a trav√©s del √°rbol de componentes sin tener que pasar props manualmente en cada nivel.

### Crear un Context

```javascript

```

### Provider: Provee el valor del contexto

```javascript

  const [datos, setDatos] = useState('valor inicial');
  
  return (
    <MiContexto.Provider value={{ datos, setDatos }}>

```

### Consumer: Consume el valor del contexto

```javascript

  const { datos, setDatos } = useContext(MiContexto);
  return <div>{datos}</div>;
}
```

## 3. Destructuring (Desestructuraci√≥n)

> Sintaxis para extraer valores de objetos y arrays de forma m√°s concisa.

### Destructuring de Objetos


## 4. Spread Operator (...)

> Expande elementos de un iterable (array u objeto).

### Con Arrays

```javascript
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];

// Combinar arrays:
const combinado = [...arr1, ...arr2];
// [1, 2, 3, 4, 5, 6]

// Copiar array:
const copia = [...arr1];
// [1, 2, 3] (nuevo array)
```

### Con Objetos

```javascript
const obj1 = { a: 1, b: 2 };
const obj2 = { c: 3, d: 4 };

// Combinar objetos:
const combinadoObj = { ...obj1, ...obj2 };
// { a: 1, b: 2, c: 3, d: 4 }

// Copiar y modificar:
const modificado = { ...obj1, b: 10 };
// { a: 1, b: 10 } (b se sobrescribe)

// Uso en setState:
setFormData(prev => ({
  ...prev,      // Copia todas las propiedades existentes
  nombre: 'Juan' // Sobrescribe/agrega 'nombre'
}));
```

## 7. Operadores L√≥gicos para Renderizado

### Operador && (AND)

> Si la izquierda es truthy, eval√∫a y retorna la derecha. Si la izquierda es falsy, retorna la izquierda (no eval√∫a la derecha).

```javascript

// Si usuario es null/undefined (falsy), no muestra nada

// Solo muestra el span si count es mayor a 0
```

### Operador || (OR)

> Si la izquierda es truthy, retorna la izquierda. Si la izquierda es falsy, eval√∫a y retorna la derecha.

```javascript

// Si cantidad es 0, null, undefined, '', usa 1

// Si no hay nombre, usa 'An√≥nimo'
```

### Operador Ternario (? :)

> `condicion ? siTrue : siFalse` - Es como un if-else en una sola l√≠nea.

```javascript

// En renderizado:
{carrito.length === 0 ? (
  <p>Carrito vac√≠o</p>
) : (
  <ListaProductos />
)}
```

### Optional Chaining (?.)

> Acceso seguro a propiedades anidadas. Si algo es null/undefined, retorna undefined sin error.

```javascript
// Sin optional chaining (puede dar error):

// Con optional chaining (seguro):

// Si usuario o contacto son null/undefined, email = undefined (sin error)

// Con llamadas a funciones:

// Solo llama a saludar() si usuario y saludar existen
```

## 8. Array Methods (M√©todos de Arrays)

## 13. Expresiones Regulares (Regex)

> Patrones para buscar y validar strings.

### Sintaxis B√°sica

### Explicaci√≥n del Patr√≥n de Email

```
/^[^\s@]+@[^\s@]+\.[^\s@]+$/

^        : Inicio del string
[^\s@]+  : Uno o m√°s caracteres que NO sean espacio ni @
@        : El s√≠mbolo @
[^\s@]+  : Uno o m√°s caracteres que NO sean espacio ni @
\.       : Un punto literal (\ escapa el punto)
[^\s@]+  : Uno o m√°s caracteres que NO sean espacio ni @

```

### Flags Comunes

- `i` = case insensitive (no distingue may√∫sculas)
- `g` = global (encuentra todas las coincidencias)
- `m` = multiline

### Validar Solo D√≠gitos
